# E2E 流程草案：下单 → 支付 → 收货

此文件为端到端用例说明草案（可用于 Playwright/Cypress 实现）。

## 前置条件
- 已有可用商品（或通过接口创建）
- 已登录并持有有效 token

## 步骤
1. 访问商品列表页，选择商品，进入详情页
2. 点击“加入购物车”，断言购物车计数增加
3. 进入购物车页，修改数量，断言金额更新
4. 点击“下单”，等待订单号生成并显示成功提示
5. 进入订单列表页，找到该订单并进入详情
6. 点击“支付”，断言状态变为“已支付”
7. 点击“确认收货”，断言状态变为“已完成”

## 接口校验（可选）
- 使用 API 验证订单状态流水（from-cart → pay → receive）

## 备注
- 若使用 Playwright：建议在 `tests/e2e` 目录初始化配置，使用 `page.route` 对登录等接口进行简化或直接真实调用。
- 若使用 Cypress：在 `cypress/e2e` 目录中编写同名用例，利用 `cy.request` 辅助 API 断言。
