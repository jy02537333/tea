## 2. 门店后台（3.3）

- **PRD 位置**：`doc/prd.md` → `3.3.2 商品管理` 中的「商品分类（服务商品/外卖商品/其他） [] — 实现时关注」。
- **当前状态**：
  - 已明确门店有独立的商品分类维度，区分「服务商品/外卖商品/其他」。
  - 尚未对分类枚举、层级（一级/二级）、与前台展示/筛选的关系给出完整表述。
- **后续建议动作**：
  - 输出一份门店商品分类设计：
    - 分类类型枚举：服务类（堂食点单）、外卖类、其他（例如会员卡、虚拟权益）。
    - 是否支持多级分类、排序、启用/停用；
    - 前端映射关系：某类商品在门店点单页/外卖页如何展示与过滤。

### 2.2 门店营销与活动管理（3.3.8） []

- **PRD 位置**：`doc/prd.md` → `3.3.8 门店营销与活动管理（补充说明） []`
- **当前状态**：
  - 已有高层描述：门店可以参与平台活动、可能有门店自建活动与门店优惠券能力；
  - 平台级营销模型在 `3.2.6 营销管理` 已完备，但门店侧「能做什么」「做多少」尚未完全裁剪清楚。
- **后续建议动作**：
  - 明确门店可用的营销能力子集：
    - 门店是否能创建仅本店可见的活动？字段是否等同平台活动？
    - 门店活动是否需要平台审核发布？
    - 门店活动数据如何汇总进平台报表？
  - 若短期只允许「门店参与平台活动」而不支持自建活动，应在 PRD 中明确写死，避免实现阶段走偏。

### 2.3 门店优惠券配置细节（门店发券能力） []

- **PRD 位置**：
  - `doc/prd.md` → `3.3.5 会员管理（门店维度）` 中门店发券相关描述；
  - 对应功能清单：5.3 中「营销管理-优惠券」。
- **当前状态**：
  - 已确定「门店可以对本店会员发券」这一能力；
  - 但尚未明确门店侧优惠券字段是否完全沿用平台优惠券模型（`3.2.6`），还是只能使用字段子集（如仅允许配置满减、有效期，不允许复杂人群/渠道规则）。
- **后续建议动作**：
  - 在门店 PRD 或营销 PRD 中补一小段：
    - 门店券字段 = 平台券字段的哪一部分；
    - 是否允许门店独立设定使用范围（仅本店 / 本店指定商品）；
    - 审核/风控策略（例如高面额券是否需要平台审批）。

### 2.4 门店财务提现/结算流程（3.3.4） []

- **PRD 位置**：`doc/prd.md` → `3.3.4 财务管理` 中的「财务提现（提现申请） [] — 实现时关注」。
- **当前状态**：
  - 平台侧财务与门店结算能力在 `3.2.7 财务管理` 已有说明；
  - 门店前台仅简要提到「财务提现入口」，尚未定义：

- **后续建议动作**：
  - 输出一页「门店结算与提现流程」：
    - 结算单生成逻辑（按自然日/自然周/自然月、或按手动发起生成）；
