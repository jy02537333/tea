启动三个服务的启动命令（适用于 Windows PowerShell）

说明：下面给出在开发机器上启动 `tea-api`（Go 后端）、`Admin-FE`（管理端静态前端）和 `WX-FE`（小程序前端）的常用命令。根据你机器环境选择合适的方式（如有 Node.js 可用，可以使用 `npx serve`；若用 Python 则用 `python -m http.server`）。

一、tea-api（后端）

# 进入服务目录
cd E:\project\tea\tea-api

# 开发模式（实时输出到控制台）
go run main.go

# 或：先编译再运行（更稳定）
go build -o tea-api.exe main.go
.\tea-api.exe

# 后台运行并重定向日志（分别到 stdout/err 文件）
Start-Process -FilePath go -ArgumentList 'run','main.go' -RedirectStandardOutput 'E:\project\tea\tea-api\server_out.log' -RedirectStandardError 'E:\project\tea\tea-api\server_err.log' -NoNewWindow

# 验证（PowerShell）：健康检查
Invoke-RestMethod -Uri 'http://localhost:8080/api/v1/health' -Method Get


二、Admin-FE（管理端前端，静态文件）

# 进入前端目录
cd E:\project\tea\Admin-FE

# 方案 A：使用 Python 内置静态服务器（免依赖）
python -m http.server 8081
# 打开浏览器访问 http://localhost:8081/

# 方案 B：如果安装了 Node.js，使用 npx 的静态服务器（更接近生产）
npx serve -l 8081 .


三、WX-FE（小程序前端）

# 注意：小程序通常使用「微信开发者工具」打开项目文件夹（GUI），不是一个简单的静态服务器。
# 在开发者工具中选择项目目录 `E:\project\tea\WX-FE`，点击编译/预览。

# 如果只是想本地预览静态资源或提供给其它调试工具（非正式小程序运行），可以临时用静态服务器：
cd E:\project\tea\WX-FE
python -m http.server 8082
# 访问 http://localhost:8082/ （仅用于查看静态文件，非小程序真机/模拟器运行）


四、组合快速启动（PowerShell 脚本示例）

# 将下面保存为 start-all.ps1 并在项目根执行，可同时启动后端与两个静态前端（以后台进程方式运行）
<#
cd E:\project\tea

五、常见问题与提示
- 如果端口冲突（例如 :8080 已被占用），使用 `Get-NetTCPConnection -LocalPort 8080` 和 `Stop-Process -Id <pid>` 查找并释放端口，或修改 `configs/config.yaml` 中 `server.port`。
- 小程序（WX-FE）最好用微信开发者工具打开项目目录进行真机/模拟器调试。
- 若希望我把 `start-all.ps1` 脚本也生成为可执行文件，我可以为你创建并测试（需要你确认是否允许在当前环境并行运行这些进程）。
